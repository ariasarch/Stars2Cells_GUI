================================================================================
STARS2CELLS PIPELINE - INPUT FILE FORMAT SPECIFICATION
================================================================================

OVERVIEW
--------
The Stars2Cells pipeline requires neuron centroid data per session. Two input
formats are accepted: a dictionary (recommended) or a raw 3D footprint matrix.
Each session is a separate .npy file with a standardized filename.


FILE NAMING CONVENTION
----------------------
Pattern (regex): ^([A-Za-z0-9_]+?)_(\d+)(.*?)\.npy$

  Group 1 (animal_id):  alphanumeric + underscores  (NON-greedy)
  Group 2 (session_id): one or more DIGITS          (required)
  Group 3 (suffix):     optional trailing text before .npy

The pipeline extracts:
  - animal_id   = group 1
  - session_name = full filename stem (without .npy)

Examples:
  ✓ 408021_758519303.npy             → animal=408021,    session=758519303
  ✓ mouse1_758519303.npy             → animal=mouse1,    session=758519303
  ✓ subj_a_758519303_postlearn.npy   → animal=subj_a,    session_name keeps the suffix
  ✓ 12345_67890.npy                  → animal=12345,     session=67890
  ✗ mouse1_session1.npy              (session portion is not all digits)
  ✗ 408021-758519303.npy             (wrong separator)
  ✗ 758519303.npy                    (no animal_id segment)

NOTE: Although animal_id may contain underscores, the regex is non-greedy and
splits at the FIRST underscore-followed-by-digits. If your animal_id ends in
digits, it can be ambiguous — prefer animal_id segments that end in a letter
or underscore (e.g. `mouse_a_758519303.npy`, not `mouse5_758519303.npy` if
you intend the `5` to be part of the animal name).


────────────────────────────────────────────────────────────────────────────────
FORMAT 1 (RECOMMENDED): DICTIONARY
────────────────────────────────────────────────────────────────────────────────

A Python dict saved with `np.save(..., allow_pickle=True)`.

REQUIRED keys:
  1. centroids_x  (array/list of N floats — X-coordinates in pixels)
  2. centroids_y  (array/list of N floats — Y-coordinates in pixels)

OPTIONAL keys (used if present, otherwise auto-generated):
  3. roi_ids     (array/list of N unique IDs — ints or strings)
                 → if missing, defaults to np.arange(N)

IGNORED keys (the pipeline does NOT read these from the dict — animal_id and
session come from the FILENAME):
  - subject_id, session_id, footprints, traces, timestamps, etc.
  Including them is harmless; they're simply not used.

Data requirements:
  ✓ len(centroids_x) == len(centroids_y)
  ✓ len(roi_ids) == len(centroids_x)  (if provided)
  ✓ N >= 4  (sessions with fewer neurons are skipped)
  ✓ Pixel coordinates (typically origin at top-left)
  ✓ ROI IDs unique within session (if provided)

Internal note: the pipeline stacks centroids as (y, x) order internally — you
still pass them in as separate centroids_x and centroids_y keys.


────────────────────────────────────────────────────────────────────────────────
FORMAT 2 (FALLBACK): RAW 3D FOOTPRINT MATRIX (A MATRIX)
────────────────────────────────────────────────────────────────────────────────

A 3D numpy array of shape (H, W, N) — the spatial footprints for N neurons.
The pipeline auto-extracts centroids via `extract_centroids_from_A` and
generates sequential roi_ids via np.arange(N).

Use this only if you already have an A matrix; Format 1 is preferred because
it's smaller and avoids re-running centroid extraction.


PYTHON CODE — FORMAT 1 (DICT)
------------------------------
import numpy as np

centroids_x = [123.4, 456.7, 789.0, 234.1]
centroids_y = [234.5, 567.8, 890.1, 345.2]
roi_ids     = [0, 1, 2, 3]   # optional

data = {
    'centroids_x': np.array(centroids_x),
    'centroids_y': np.array(centroids_y),
    'roi_ids':     np.array(roi_ids),   # optional
}

# Filename: animal_id must end before the first `_<digits>` segment
np.save('408021_758519303.npy', data, allow_pickle=True)


LOADING AND VERIFYING A FILE
-----------------------------
import numpy as np
from pathlib import Path
import re

path = Path('408021_758519303.npy')

# 1. Filename check
assert re.match(r'^([A-Za-z0-9_]+?)_(\d+)(.*?)\.npy$', path.name), "Bad filename"

# 2. Load
raw = np.load(path, allow_pickle=True)

# 3. Format 1 (dict) or Format 2 (3D A matrix)?
if isinstance(raw, np.ndarray) and raw.ndim == 0:
    data = raw.item()
    assert isinstance(data, dict), "0-d array did not contain a dict"
    assert 'centroids_x' in data and 'centroids_y' in data, "Missing centroids"
    n = len(data['centroids_x'])
    assert len(data['centroids_y']) == n, "centroids_y length mismatch"
    if 'roi_ids' in data:
        assert len(data['roi_ids']) == n, "roi_ids length mismatch"
    print(f"✓ Format 1 (dict): {n} neurons")

elif isinstance(raw, np.ndarray) and raw.ndim == 3:
    n = raw.shape[2]
    print(f"✓ Format 2 (A matrix): {n} neurons, shape {raw.shape}")

else:
    raise ValueError("Unrecognised file format")

assert n >= 4, f"Need at least 4 neurons, found {n}"


VALIDATION CHECKLIST
--------------------
□ Filename matches: ^([A-Za-z0-9_]+?)_(\d+)(.*?)\.npy$
□ File loads via np.load(..., allow_pickle=True)
□ Either a dict (Format 1) or a 3D ndarray (Format 2)
□ For Format 1: centroids_x and centroids_y both present, equal length
□ For Format 1: if roi_ids present, length matches centroids
□ N >= 4 neurons


COMMON ERRORS
-------------
"Not enough neurons (N < 4)"
  → Session has fewer than 4 neurons; will be skipped.

"NO MATCH: filename.npy"
  → Filename doesn't match the regex. Check that the session segment is all
    digits and that segments are separated by underscores.

"Unrecognised file format"
  → File is neither a dict (in a 0-d ndarray) nor a 3D ndarray. Check that
    you saved the dict with allow_pickle=True.

KeyError: 'centroids_x'  or  KeyError: 'centroids_y'
  → Format 1 dict is missing a required centroid array.


DIRECTORY STRUCTURE
-------------------
input_directory/
├── 408021_758519303.npy
├── 408021_758519304.npy
├── 408021_758519305.npy
└── 412345_758519400.npy

Run with:
    python run_step_1_parallel.py --input-dir input_directory/

Optional filter to a single animal:
    config.animal_id = "408021"   # only sessions where group(1) == "408021"


CONVERTING FROM OTHER FORMATS
------------------------------
From MATLAB / Suite2P:
  Extract stat.mat or F.npy with ROI coordinates → build centroids_x/y arrays
  → save as Format 1 dict.

From CaImAn:
  Use cnm.estimates.coordinates → centroids_x/y → Format 1 dict.
  Or save cnm.estimates.A reshaped to (H, W, N) → Format 2.

From EXTRACT:
  Parse output.mat for centroids → Format 1 dict.


NOTES
-----
- The pipeline internally stacks centroids as (y, x); callers still pass
  centroids_x and centroids_y as separate keys.
- Extra dict keys are allowed and ignored (subject_id, session_id, footprints,
  traces, timestamps, etc.).
- ROI IDs don't need to be sequential, but must be unique within a session.
- All units are pixels; no coordinate transformation is applied.
- Use a consistent coordinate system across sessions for the same animal.
